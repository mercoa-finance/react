# PayableApproversV1

The `<PayableApproversV1>` part manages the selection of approvers for an invoice. It provides a flexible interface for assigning users or roles to approval slots, supporting both default UI and deep customization. This part is typically used within approval workflows in the Mercoa platform.

## Props

| Name         | Type                                                                                   | Required | Description                                                                                                 |
| ------------ | -------------------------------------------------------------------------------------- | -------- | ----------------------------------------------------------------------------------------------------------- |
| `invoiceId`  | `string`                                                                              | ✅       | The ID of the invoice for which you assign approvers.                                               |
| `onChange`   | `(approvers: ApproverSelection[]) => void`                                             | ✅       | Callback triggered when the list of assigned approvers changes.                                             |
| `readOnly`   | `boolean`                                                                             |          | If true, disables editing; the part displays in read-only mode.                                             |
| `renderCustom` | `{ resetSelection?: (resetSelection: () => void) => React.ReactNode }`               |          | Provides custom UI rendering hooks. See ["Custom Rendering (renderCustom)"](#custom-rendering-renderCustom) |

---

### Custom Rendering (`renderCustom`)

The `renderCustom` prop enables you to override or extend portions of the UI within the approver selection interface. Now, the following customization point is available:

| Property          | Type                                        | Description                                                                                                                                |
| ----------------- | ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `resetSelection`  | `(resetSelection: () => void) => ReactNode` | A function that receives the internal `resetSelection` callback. Use this to render a custom "Reset Selection" button or control.           |

#### How it works

- The `resetSelection` property is a function you give. `PayableApproversV1` calls it with a callback as its argument.
- Your function should return a React node (for example, a button). When your UI element calls the provided `resetSelection` function, the approver selection resets to its initial state.
- This allows you to give a custom location, style, or logic for the reset action within the selection interface.

**This prop is optional** and doesn't affect the default UI unless you give a custom implementation.

## Usage Example

### Custom Reset Selection Button

```jsx
import { PayableApproversV1 } from '@mercoa/react';

<PayableApproversV1
  invoiceId="inv_123"
  onChange={(approvers) => {
    // handle updated approver selection
  }}
  renderCustom={{
    resetSelection: (resetSelection) => (
      <button
        type="button"
        onClick={resetSelection}
        style={{
          background: '#f5f5f5',
          border: '1px solid #ccc',
          padding: '6px 12px',
          borderRadius: '4px',
          cursor: 'pointer',
          marginBottom: '12px',
        }}
      >
        Reset Approver Selection
      </button>
    ),
  }}
/>
```
