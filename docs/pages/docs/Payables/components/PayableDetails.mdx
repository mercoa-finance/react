# Payable Details

The `<PayableDetails>` component is used to let users create and manage payable invoices.

## Customization Options

| Themeable | Variations | Custom Component |
| :-------: | :--------: | :--------------: |
|    ✅     |     ✅     |        ✅        |

## Props

### Main Props

| Name             | Type                                                      | Required | Description                                                                                  |
| ---------------- | --------------------------------------------------------- | -------- | -------------------------------------------------------------------------------------------- |
| `queryOptions`   | [`PayableDetailsQueryOptions`](#data-options)             |          | Data-related options such as `invoiceId` and `invoiceType`.                                  |
| `handlers`       | [`PayableDetailsHandlers`](#handlers)                     |          | Callback functions for handling invoice and counterparty submissions.                        |
| `config`         | [`PayableDetailsConfig`](#functional-options)             |          | Functional configurations like `supportedCurrencies`.                                        |
| `displayOptions` | [`PayableDetailsDisplayOptions`](#display-options)        |          | Options to adjust layout and visibility settings like `heightOffset` and `documentPosition`. |
| `renderCustom`   | [`PayableDetailsRenderCustom`](#custom-rendering-options) |          | Custom rendering options for specific components such as toast notifications.                |
| `children`       | `ReactNode`                                               |          | Allows rendering custom children components.                                                 |

## PayableApproversV1

The `<PayableApproversV1 />` component (aliased as `<PayableApprovers />`) renders the invoice approvers section and can appear as part of custom children in `<PayableDetails />`. 

**New in v0.1.5-rc3:**  
The component now supports three new optional props:

| Prop               | Type      | Default         | Description                                                                          |
|--------------------|-----------|-----------------|--------------------------------------------------------------------------------------|
| `allowAnyApprover` | `boolean` | `true`          | If `true`, the "Any Approver" option appears in the approver selector.               |
| `title`            | `string`  | `'Approvals'`   | Section heading text for the approvers area.                                         |
| `selectorLabel`    | `string`  | `'Assigned to'` | Label text for the approver selector combo box.                                      |
| `readOnly`         | `boolean` | `false`         | Renders the part in a read-only state (existing prop, unchanged).                    |

**Usage Notes:**
- If `allowAnyApprover={false}`, the "Any Approver" option is not shown in the selector.
- Use `title` and `selectorLabel` to customize UI text.
- All props are optional; defaults are used if not specified.

---

## PayableApproversV1 Usage Examples

### Example 1: Hide the "Any Approver" Option

```jsx
<PayableApprovers allowAnyApprover={false} />
```

### Example 2: Customizing Section Title and Selector Label

```jsx
<PayableApprovers
  title="Invoice Approval Team"
  selectorLabel="Reviewer"
/>
```

### Example 3: Full Customization

```jsx
<PayableApprovers
  allowAnyApprover={false}
  title="Custom Approvers Section"
  selectorLabel="Select Approver"
/>
```
