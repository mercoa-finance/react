# Approval Policy History

## Overview

The Approval Policy component now supports browsing and restoring approval policy history. This feature allows users with permissions to:

- View the history of approval policies
- Inspect previous versions with their triggers and rules
- Restore a previous version as the active policy

---

## ApprovalPolicy Props

| Name               | Type                              | Default   | Description                                                         |
| :----------------- | :------------------------------- | :-------- | :------------------------------------------------------------------ |
| showHistory        | `boolean`                        | `false`   | Show the approval policy history dialog and enable restoration.      |
| approvalPolicies   | `Mercoa.ApprovalPolicyResponse[]` |           | (Optional) Specify policies to display; otherwise fetched internally |
| readOnly           | `boolean`                        | `false`   | If true, disables editing and restoring; view-only mode.             |

### Prop Behaviors

- **showHistory**: When true, users can open a dialog to browse and restore previous approval policy versions.
- **approvalPolicies**: If set, these policies are shown directly; otherwise, policies are fetched automatically.
- **readOnly**: When true, the policy and history dialog are view-only and restoration is disabled.

---

## Usage Example

```jsx
<ApprovalPolicy
  entityId="ent_XXXXX"
  showHistory={true}
/>
```

---

## Error Handling and Permissions

- Accessing or restoring approval policy history requires proper authentication and authorization.
- If there is no history, the dialog indicates no previous versions are available.
- If restore fails or permissions are insufficient, an error message is displayed.
- The following HTTP error codes may apply: 401 Unauthorized, 403 Forbidden, 404 Not Found, 409 Conflict, 422 Unprocessable Entity.
