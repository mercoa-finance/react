# Payable Details

The `<PayableDetailsV1>` component is used to let users create and manage payable invoices.

## Props

| Name                  | Type                                                                                 | Required | Description                                                                                                                                           |
| :---------------------| :------------------------------------------------------------------------------------| :--------| :-----------------------------------------------------------------------------------------------------------------------------------------------------|
| invoiceId             | `string`                                                                             |          | The ID of the invoice to edit. Leave blank if creating a new invoice                                                                                  |
| onUpdate              | `(invoice: InvoiceResponse \| undefined) => void`                                    |          | When the invoice is created, updated, or deleted, this callback is triggered                                                                          |

## Payable Approvers

The `<PayableApproversV1>` part assigns approvers to an invoice as part of approval policies within invoice flows.

### Props

| Name         | Type                                                                                                              | Required | Description                                                                                                        |
| :----------- | :--------------------------------------------------------------------------------------------------------------- | :--------| :------------------------------------------------------------------------------------------------------------------|
| renderCustom | `{ resetSelection?: (resetSelection: () => void) => React.ReactNode }`                                           |          | Optional. Allows injection of custom UI into the approver selection interface. Use `resetSelection` to give a custom reset button or logic. |

**Type signature:**

```typescript
renderCustom?: {
  resetSelection?: (resetSelection: () => void) => React.ReactNode
}
```

#### Example Usage

```jsx
<PayableApproversV1
  /* ...other props */
  renderCustom={{
    resetSelection: (resetSelection) => (
      <button
        type="button"
        style={{ marginLeft: 8, color: 'red', background: 'none', border: 'none', cursor: 'pointer' }}
        onClick={resetSelection}
      >
        Clear All Approvers
      </button>
    ),
  }}
/>
```

In this example, a custom "Clear All Approvers" button renders in the approver selection interface. Clicking it calls the internal `resetSelection` method to clear all selected approvers.

## Customization

The `PayableDetails` consists of the [`<PayableFormV1>`](./Form) and [`<PayableDocumentV1>`](./Document) components. See those pages for how to customize them.

The default children is the following:

```tsx
<PayableDetailsV1>
 {(props) => [
     <PayableDocumentV1 {...props} key={1} />,
     <PayableFormV1 {...props} key={2} />
  ]}
</PayableDetailsV1>
```
