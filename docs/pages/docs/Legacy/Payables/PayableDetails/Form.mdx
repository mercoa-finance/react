# Payable Form

Use the `<PayableForm>` part as a child of the [`<PayableDetails>`](./Details) part. Normally, the parent `<PayableDetails>` part provides all props. It renders the form used to display and update the payable data.

| Name        | Type                                                                                                                                                               | Required | Description                                                                                   |
| :---------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------- | :-------------------------------------------------------------------------------------------- |

| renderCustom | `{ resetSelection?: (resetSelection: () => void) => React.ReactNode }`                                                                                            |          | Optional. Allows injection of a custom UI (for example, a button) that, when triggered, calls the internal selection reset function. |

#### About `renderCustom.resetSelection`

The `renderCustom` prop lets you give a custom part (such as a button or link) to reset the selection of approvers in the selection interface. Pass an object with a `resetSelection` function, which receives an internal callback as its argument. Calling this callback resets the internal selection state.

**Usage Example:**

```jsx
<PayableFormV1 invoice={inv_new_ready}>
  {() => (
    <PayableApproversV1
      renderCustom={{
        resetSelection: (resetSelection) => (
          <button type="button" onClick={resetSelection} style={{ marginLeft: 8 }}>
            Custom Reset Selection
          </button>
        ),
      }}
    />
  )}
</PayableFormV1>
```

- The `resetSelection` function provided by Mercoa clears the current selection when called.
- Your implementation should return a ReactNode (for example, a button, link, or icon).

<Callout>
  Using <code>renderCustom</code> doesn't override other existing props or behaviors. It allows you to add more UI controls for actions like resetting the selection. The internal state and error handling remain managed by the part.
</Callout>
