#### ApproversSelectionV1

The `ApproversSelectionV1` component allows users to select approvers for a payable. Starting with version `0.1.11-rc1`, it includes enhanced customization for the reset selection UI.

##### Props

| Name         | Type                                                                                                                                   | Required | Description                                                                                                                                              |
| :----------- | :------------------------------------------------------------------------------------------------------------------------------------- | :------- | :------------------------------------------------------------------------------------------------------------------------------------------------------- |
| allowAnyApprover | `boolean`                                                                                                                          |          | If true, allows any approver to be selected. Defaults to true.                                                                                           |
| label        | `string`                                                                                                                               |          | Custom label for the selection UI.                                                                                                                       |
| renderCustom | `{ resetSelection?: (resetSelection: () => void) => React.ReactNode }`                                                                 |          | Allows custom rendering of the "Reset Selection" UI. If provided, the default "Reset Selection" option is hidden and your custom UI is shown instead.    |

###### renderCustom.resetSelection

- **Type:** `(resetSelection: () => void) => React.ReactNode`
- **Description:**  
  If provided, this function will be called with a `resetSelection` handler. You can use this handler in your custom UI (e.g., a button or link) to trigger reset behavior.

- **Behavior:**
  - If `renderCustom.resetSelection` is **not** provided, the default "Reset Selection" UI is displayed.
  - If `renderCustom.resetSelection` **is** provided, the default UI is hidden and your custom UI is rendered instead.

##### Usage Examples

###### Default Behavior

```jsx
<ApproversSelectionV1 allowAnyApprover label="Select Approvers" />
```

This renders the default selection UI, including the default "Reset Selection" option.

###### Custom Reset Selection UI

You can supply a custom reset UI using the `renderCustom` prop. The provided function receives a `resetSelection` callback you should call to trigger the reset.

```jsx
<ApproversSelectionV1
  label="Custom Approvers"
  renderCustom={{
    resetSelection: (resetSelection) => (
      <button
        type="button"
        onClick={resetSelection}
        style={{
          background: 'transparent',
          border: 'none',
          color: '#8855FF',
          cursor: 'pointer',
          marginTop: 8,
        }}
      >
        <span role="img" aria-label="refresh">
          ðŸ”„
        </span>{' '}
        Clear Approver Selection
      </button>
    ),
  }}
/>
```

> **Version Note:**  
> The `renderCustom` prop for customizing the reset selection UI in `ApproversSelectionV1` is available as of `@mercoa/react` version **0.1.11-rc1**.
